DNS Troubleshooting Case Study

This project demonstrates how I diagnosed and resolved a DNS configuration issue that was preventing a remote employee from accessing internal company websites. Internet access was functional, but internal tools remained unreachable due to incorrect DNS settings.

Objective

  • Identify causes of internal domain resolution failures
  • Restore access to internal tools while maintaining internet connectivity
  • Improve reliability of DNS settings for remote users

Symptoms

  • Public websites like Google and YouTube loaded normally
  • Internal tools like intranet.company.local failed to load
  • ping and nslookup for internal domains timed out

Investigation

  1. Confirmed internet connectivity was working
  2. Ran nslookup intranet.company.local – response: “DNS request timed out”
  3. Checked settings with ipconfig /all – DNS was set to 8.8.8.8
  4. Pinged internal DNS server at 10.0.0.5 – successful

Resolution

  • Updated DNS server address to 10.0.0.5 in network settings
  • Flushed local DNS cache using: 
    ipconfig /flushdns
  • Verified success with: 
    nslookup intranet.company.local
  • Internal tools loaded successfully

Lessons Learned

  • DNS issues can block internal resources without affecting general browsing
  • nslookup is a powerful tool to diagnose DNS failures
  • Remote employees should use company DNS servers for full functionality
